## Project Overview

**Typed Context via Persistent Rotation (TCPR)** — a research project investigating typed context encoding via RoPE (Rotary Positional Embeddings) rotation to defend against prompt injection attacks in LLMs. The core idea: encode token-level source metadata (system vs user vs external) through rotary transformations in underutilized RoPE frequency dimensions, enabling source-aware attention. Target model: Meta-Llama 3.1 8B Instruct.

## Architecture

### Module Layout

- **`model/`** — Core typed RoPE implementation
  - `typed_rope.py`: Pure tensor operations — `create_type_rotation()` generates cos/sin for type encoding in target subspaces; `apply_typed_rope()` blends position RoPE with type rotation
  - `hook_attention.py`: `TypedRoPEHooks` class installs PyTorch forward hooks on all attention layers at inference time (non-invasive patching)
- **`utils.py`** — Shared utilities: model loading (fp16/int8/int4), config loading, chat template tokenization, token source labeling (`assign_source_labels()` for 2-type, `assign_source_labels_3type()` for 3-type)
- **`data/`** — Dataset generators producing `.jsonl` files (normal conversations, PI attacks across 4 categories: extraction, override, role_play, smuggling)
- **`analysis/`** — Phase 1-2 scripts: hidden state extraction, linear probe training, RoPE frequency spectrum analysis, dimension ablation
- **`experiments/`** — Phase 3-4 scripts: PPL sweep, attention analysis, ICL experiments (5 conditions), quantization checks, LoRA training/evaluation
- **`configs/`** — YAML model config (`llama8b.yaml`), plus generated configs for experiment parameters and target subspaces
- **`outputs/`** — Generated figures (Fig 1-12) and results summary

### Key Design Patterns

- **Hook-based patching**: Type rotation injected via PyTorch forward hooks on rotary embedding modules — no model weight modification needed
- **Persistent signal**: Type rotation applied at every attention layer through RoPE, so it cannot be diluted through network depth
- **Synthetic mode**: Every experiment script supports `--synthetic` for CPU-only validation without loading the full model
- **Type rotation formula**: Standard RoPE uses `angle = position * frequency`; Typed RoPE replaces this with `angle = type_id * rotation_angle` in target subspaces only

### 4-Phase Research Design

1. **Probing** — Detect existing source encoding in hidden states via linear probes
2. **RoPE Analysis** — Identify underutilized RoPE dimensions (low-frequency subspaces with wavelength >> max context) as targets
3. **ICL Verification** — Test typed RoPE with in-context learning across 5 conditions (baseline, delimiter, rotation-only, rotation+ICL, random-rotation control)
4. **LoRA Finetuning** — Amplify type signal via parameter-efficient finetuning with typed RoPE hooks active

### Configuration

Model parameters are in `configs/llama8b.yaml` (head_dim=128, 32 layers, 32 attention heads, GQA with 8 KV heads, rope_theta=500000). Target subspaces are generated by `analysis/rope_ablation.py` into `configs/target_subspaces.json`. Experiment parameters (max_safe_angle, chosen_num_subspaces) go into `configs/experiment.yaml`.
